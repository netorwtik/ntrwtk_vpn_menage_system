# VPN Payment Manager Bot

Telegram-бот для учёта подтверждённых оплат пользователей небольшого VPN-сервиса.
Переводы выполняются вне бота, а администратор фиксирует в системе только проверенные
поступления.

## Статус

Проект содержит рабочий MVP:

- настроены Node.js, TypeScript, ESLint и Prettier;
- подготовлены каталоги для слоёв приложения;
- добавлены Dockerfile, docker-compose и шаблон `.env`;
- PostgreSQL готов к локальному запуску в контейнере;
- добавлены Prisma-схема, первая миграция и Prisma Client;
- подключены Telegram polling, проверка администратора и команды `/start` и `/help`.
- реализованы добавление пользователя `/add_user` и список пользователей `/users`.
- реализованы изменение тарифа `/set_price` и статуса `/set_status`.
- реализованы ручное подтверждение оплаты `/paid` и история оплат `/history`.
- реализованы контроль задолженностей `/debtors` и ручной список напоминаний `/remind`.
- реализованы одноразовое подключение пользователя `/invite` и пользовательский статус.
- реализована кнопка пользователя `Я оплатил` с подтверждением или отклонением заявки
  администратором.
- реализована админская панель `/admin` с кнопочным просмотром заявок, должников,
  ближайших оплат и пользователей.
- реализована пользовательская карточка оплаты с реквизитами и отправкой заявки через
  inline-кнопку.

## Границы MVP

- доступ к административным командам только для Telegram ID из конфигурации;
- управление пользователями, тарифами и статусами;
- ручное подтверждение администратором фактически полученных оплат;
- история подтверждённых оплат, должники и список для напоминаний;
- сообщения бота на русском языке.

Заявка пользователя не продлевает доступ автоматически: подтверждение выполняется
администратором после проверки поступления на карте. В дальнейшем будет добавлена
автоматическая отправка напоминаний.

Модуль `payment-claims` реализует подтверждаемую администратором заявку пользователя
`Я оплатил`. Автоматическая доставка напоминаний остаётся следующим этапом развития.

## Модель данных MVP

- `User` - пользователь VPN, тариф, статус, дата начала пользования и дата оплаченного доступа.
- `Payment` - только подтверждённый администратором перевод, включая сумму, период оплаты,
  способ перевода и Telegram ID подтвердившего администратора.
- `PaymentSource.ADMIN_MANUAL` - ручная фиксация оплаты администратором в MVP.
- `PaymentSource.CLAIM_CONFIRMED` - платёж, созданный после подтверждения заявки
  пользователя `Я оплатил`.

## Доступные команды

На текущем этапе бот принимает команды только от Telegram ID из `ADMIN_TELEGRAM_IDS`:

```text
/start - проверить административный доступ
/admin - открыть панель администратора
/help  - показать справку
/add_user @username Имя | тариф | комментарий - добавить пользователя
/users - показать всех пользователей
/set_price @username тариф - изменить тариф
/set_status @username active|paused|disabled - изменить статус
/paid @username сумма способ | комментарий - подтвердить полученную оплату
/history @username - показать историю подтверждённых оплат
/debtors - показать активных пользователей с просроченной оплатой
/remind - показать просрочки и ближайшие окончания оплаты
/invite @username - создать персональную ссылку подключения пользователя
```

Пример добавления:

```text
/add_user @ivan Иван Петров | 300 | Оплата по СБП
```

`@username` используется как уникальный ключ пользователя и сохраняется в нижнем регистре.
Комментарий необязателен:

```text
/add_user @petr Пётр | 400
```

Управление созданным пользователем:

```text
/set_price @ivan 400
/set_status @ivan paused
```

Статусы:

- `active` - пользователь учитывается в будущих списках оплаты и напоминаний;
- `paused` - обслуживание временно приостановлено;
- `disabled` - пользователь отключён, но его история сохраняется.

Подтверждение оплаты:

```text
/paid @ivan 400 sbp | Перевод на карту
/paid @ivan 800 card | За два месяца
/history @ivan
```

- `/paid` фиксирует только уже проверенный администратором перевод; бот не принимает деньги.
- Сумма должна быть кратна текущему месячному тарифу.
- Если оплаты ранее не было или срок истёк, период начинается с даты подтверждения.
- Если доступ уже оплачен, новый период начинается со следующего дня после `paid_until`.

Контроль оплат:

```text
/debtors
/remind
```

- В списки входят только пользователи со статусом `active`.
- `/debtors` выводит просрочки и ближайшую сумму к оплате по персональному месячному тарифу.
- `/remind` включает просрочки, пользователей без первой оплаты и оплаты, срок которых
  заканчивается в пределах `REMIND_DAYS_BEFORE` дней.

При обращении с другого Telegram ID бот отвечает, что прав недостаточно.

## Панель администратора

Команда `/admin` открывает кнопочную панель для ежедневной работы:

```text
Заявки на оплату
Должники | Скоро платить
Пользователи
```

Панель показывает количество пользователей, должников, ближайших оплат и заявок, которые
нужно подтвердить. Переходы между разделами редактируют одно сообщение в чате администратора,
а не создают цепочку служебных ответов.

В разделе заявок можно открыть карточку перевода и нажать `Подтвердить оплату` либо
`Отклонить`. После решения карточка обновляется, а кнопка `В панель администратора`
возвращает к общей сводке. Команды остаются доступны для быстрых ручных действий.

## Подключение пользователя

Telegram-бот не может первым написать пользователю. Администратор создаёт одноразовую ссылку:

```text
/invite @ivan
```

После перехода пользователя по ссылке бот связывает его Telegram ID с VPN-профилем. Сырой
токен приглашения не хранится в базе: сохраняется только его хеш. Ссылка используется один
раз и по умолчанию действует `INVITE_EXPIRES_HOURS=168` часов.

После подключения пользователю доступна команда и кнопка:

```text
/status
Мой статус
```

Бот показывает статус доступа, персональный тариф, дату окончания оплаченного периода,
количество оставшихся дней либо просрочку и сумму к оплате.

## Заявка об оплате

После привязки у пользователя доступны кнопки:

```text
Мой статус
Оплата
```

Кнопка `Оплата` открывает карточку с персональной суммой, датой оплаченного доступа,
контактом администратора и реквизитами, если соответствующие значения указаны в `.env`.
В карточке расположена inline-кнопка `Я оплатил`.

При нажатии `Я оплатил` в карточке:

1. Создаётся заявка на сумму персонального месячного тарифа пользователя.
2. Администратор получает уведомление с кнопками `Подтвердить` и `Отклонить`.
3. Карточка пользователя редактируется и показывает, что заявка ожидает проверки.
4. До нажатия `Подтвердить` оплаченный период пользователя не меняется.
5. После подтверждения создаётся платёж с источником `claim_confirmed`, продлевается доступ
   и пользователь получает сообщение об успешной проверке.
6. При отклонении доступ не продлевается, пользователь получает уведомление.

Повторное нажатие `Я оплатил`, пока предыдущая заявка ожидает проверки, не создаёт дубликат.
Ежедневные автоматические напоминания о просрочке будут подключены следующим этапом; заявка
сама по себе не должна их отключать.

## Конфигурация

Секреты не хранятся в репозитории. Создайте локальный файл `.env` из шаблона:

```powershell
Copy-Item .env.example .env
```

Перед запуском приложения потребуется заполнить:

```dotenv
BOT_TOKEN=replace_with_new_bot_token
ADMIN_TELEGRAM_IDS=123456789
POSTGRES_PASSWORD=change_me
SUPPORT_USERNAME=@your_username
PAYMENT_RECIPIENT=Имя Фамилия
PAYMENT_DETAILS=СБП по номеру +7 900 000-00-00, Банк
```

- `BOT_TOKEN` - токен, полученный от BotFather. Если токен был опубликован в сообщениях,
  логах или коммитах, его необходимо отозвать через BotFather и выпустить новый.
- `ADMIN_TELEGRAM_IDS` - числовые Telegram ID администраторов через запятую.
- `POSTGRES_PASSWORD` - пароль базы; до первого запуска замените `change_me` и тем же
  значением обновите пароль внутри `DATABASE_URL`.
- `POSTGRES_PORT` - порт базы на хост-машине, по умолчанию `55432`.
- `TZ` - часовой пояс расчётов дат, по умолчанию `Europe/Moscow`.
- `REMIND_DAYS_BEFORE` - количество дней до окончания оплаты для будущих напоминаний.
- `INVITE_EXPIRES_HOURS` - срок действия ссылки привязки пользователя, по умолчанию `168`.
- `SUPPORT_USERNAME` - Telegram username администратора для связи из карточки оплаты.
- `PAYMENT_RECIPIENT` - имя получателя перевода.
- `PAYMENT_DETAILS` - реквизиты или пояснение для перевода, например номер СБП и банк.

`DATABASE_URL` из `.env.example` предназначен для команд Prisma, запускаемых с хост-машины.
Внутри Docker Compose сервис бота обращается к базе по имени контейнера `postgres`.
По умолчанию контейнер публикует базу на `127.0.0.1:55432`, чтобы не конфликтовать
с локальными установками PostgreSQL на стандартных портах.

Если `.env` был создан до смены порта, обновите в нём подключение:

```dotenv
POSTGRES_PORT=55432
DATABASE_URL=postgresql://vpn_bot:ВАШ_ПАРОЛЬ@127.0.0.1:55432/vpn_payments?schema=public
```

## Требования

- Node.js 22 или новее
- npm 10 или новее
- Docker Engine с Docker Compose v2

```powershell
npm run build
```

```powershell

```

```powershell

```

```text
/start
/help
/users
```
